Business Scenario Challenge: Online Course Enrollment
Youâ€™re building a system to record student enrollments for online courses.

Requirements:

Each enrollment has a unique Enrollment ID.
Store Student Name and Student Email.
Store Course Name and Course Start Date.
Store Enrollment Date.
Enrollment Status can be one of: Enrolled, Completed, Dropped.
Store the final Grade (optional, only if Status is Completed).

Constraints:

Enrollment Date cannot be after Course Start Date.
Grade must be between 0 and 100 if present.
Status must be one of the predefined values.
If Status is Completed, Grade must be present; otherwise, it must be null.
Student Email must be unique per Course (a student can enroll only once per course).

Reports:

1. List all students currently enrolled (Status = Enrolled) grouped by Course Name.
Include Student Name, Email, Enrollment Date.
Completed Courses with Grades

2. Show all students who have completed a course (Status = Completed).
Include Course Name, Student Name, Grade, and Course Start Date.
Dropout Rate per Course

3. Calculate the number and percentage of students who dropped a course (Status = Dropped) for each course.
Average Grade per Course

4. Calculate the average final grade for each course where students have completed the course.
Enrollment Trends Over Time

5. Show the count of enrollments per month or quarter based on Enrollment Date.
Students with Multiple Enrollments

6. Identify students enrolled in more than one course at the same time.
